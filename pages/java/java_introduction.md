## Java Introduction

Java언어에 대해서 알아보겠습니다.
<br><br>

가장 먼저 Java라는 언어의 특징을 알아보도록 하죠.
<br>

* 운영체제에 독립적입니다. `JVM`이라는 것을 통해 가능합니다. Java로 작성된 프로그램은 
OS나 하드웨어가 아닌 JVM하고만 통신하고 JVM이 Java 프로그램으로부터 전달받은 명령을
해당 운영체제가 이해할 수 있도록 변환해서 전달합니다. 따라서 Java언어는 운영체제에
독립적이만 JVM 자체는 운영체제에 종속적입니다.

{% include image.html
file='java-platform-independence.png'
%}
<br>

* 객체지향언어입니다. 객체지향개념을 가장 잘 대변하고 있는 프로그래밍 언어입니다.
* `C` 계열의 언어보다는 배우기 쉽습니다. 물론 객체지향을 이해해야 한다는 점이 있지만 
언어적인 특성상 다른 언어보다 배우기 쉬운것도 사실입니다.
* `Garbage Collection`을 이용한 자동메모리 관리가 됩니다. 개발자는 메모리에 대한 특별한
처리를 하지 않아도 사용하지 않는 메모리는 자동적으로 처리됩니다. 하지만 이 말이 프로그램을
비효율적으로 막 작성해도 된다는 의미는 아닙니다.
* 네트워크와 분산처리를 위한 다양한 API가 제공됩니다. 이로 인해 네트워크 관련 프로그램을
네트워크 이론에 대한 심도있는 이해 없이도 쉽고 빠르게 작성할 수 있습니다.
* `multi-thread`를 언어차원에서 지원합니다. 다른언어도 물론 Thread를 지원하지만 Java는
프로그래밍 언어차원에서 멀티 쓰레드를 구현하기 위한 API를 제공합니다.
* `dymanic loading`(동적 로딩)을 지원합니다. Java로 작성된 어플리케이션은 보통 여러 개의
`class`로 구성되는데 Java는 `dynamic loading`을 지원하기 때문에 실행 시에 모든 class를
로딩하지 않고 필요한 시점에 클래스를 로딩해서 사용할 수 있습니다.

## JVM

`JVM`은 `Java Virtual Machine`의 줄임말로 자바프로그램을 실행하기 위한 가상 머신입니다. 즉,
프로그램(소프트웨어)란 것이죠. 자바로 작성된 모든 프로그램은 이 `JVM`에서만 실행되기 때문에 반드시
필요한 존재입니다.
<br><br>

Java 프로그램과 일반 프로그램의 비교를 보면 다음과 같습니다.
<br>

{% include image.html
file='java-jvm-other-program-different.png'
%}
<br>

위의 그림에서 보듯이 Java 프로그램은 `JVM`을 거쳐서 실행됩니다. 우리 소스코드는 컴파일을 거쳐
`bytecode`로 만들어지고 이를 다시 JVM이 읽어들여서 interpret(해석)해서 실행시키게 됩니다.
<br><br>

하드웨어와 운영체제에 독립적이라는 독립적이라는 장점이 있지만 속도가 상대적으로 느리다는 단점이
있는데 `JIT compiler`와 향상된 최적화 기술 덕분에 속도에 아주 민감한 영역을 제외하고는
속도의 격차가 그리 크게 나지 않습니다.
<br><br>

`JIT compile`이라고 불리는 `Just-In-Time Compilation` 혹은 `dynamic translation`(동적 변환)
은 프로그램을 실행시키는 시점에 기계어로 변역하는 컴파일 기법 중 하나입니다.
<br><br>

우리가 잘 알다시피 컴퓨터 프로그램을 만드는 방식에는 컴파일 방식과 인터프리트 방식이 있는데
컴파일 방식은 우리가 작성한 프로그램을 머신 코드로 직접 번역하여 해당 머신코드를 OS 레벨에서 실행시키는
것이고 인터프리트 방식은 원시 코드 명령을 한번에 한 줄 씩 읽어들여 실행합니다. 이 때 인터프리터는 개발자가
작성한 고급 명령어들을 중간 형태로 번역한 다음 그것들을 실행하는 구조입니다.
<br><br>

Java는 그 두 방식의 혼합방식이라고 할 수 있습니다. 중간 코드 형태인 `bytecode`가 미리 생성되고 이를 런타임시에
읽어들여서 빠르게 최적화된 머신 코드로 번역해 실행시킵니다.

## JDK, JRE, JVM

`JDK`, `JRE`, `JVM`이란 용어는 많이 들어봤을 텐데 그 용어들이 무엇을 의미하고 있는지 알아보고 프로그램 환경을
구축해 보도록 하겠습니다.
<br><br>

아래 그림 하나로 설명이 될 듯 합니다.
<br>

{% include image.html
file='java-jvm-jre-jdk.png'
%}

우리는 개발을 진행해야 하므로 `JDK` 설치가 필요합니다. `JDK`안에는
다음과 같은 주요 실행파일들이 들어 있습니다.
<br>

* `javac.exe`: 자바 컴파일러, 소스코드를 바이트코드로 컴파일합니다.
* `java.exe`: 자바 인터프리터, 컴파일러가 생성한 바이트코드를 해석하고 실행합니다.
* `javadoc.exe`: 자동문서 생성기, 소스코드에 있는 주석을 이용하여 Java API와 같은
형식의 문서를 만들어 줍니다.
* `jar.exe`: 압축프로그램, 클래스파일과 프로그램의 실행에 관련된 파일들을 하나의 파일로
압축하거나 해제할 때 사용됩니다.
<br><br>

`JDK`의 버전은 `1.5`버전부터 `Java 5.0`이라고 부릅니다. `1.7`버전은 `Java 7.0`이라고 부르겠지요.
설치할 버전은 1.8(`Java 8.0`) 혹은 1.11(`Java 11`) 버전을 이용합니다. 현 시점에서 현업 프로젝트는
`Java 8.0`을 많이 이용하고 있지만 점점 `Java 11.0`으로 바뀌어 가고 있습니다. 조만간 `Java 11`을 기준으로
한 프로젝트가 더 많아질거란 보고서도 있구요.
<br><br>

우리는 `Java 11.0`을 사용하고 개발툴은 `Eclipse`를 사용하도록 하겠습니다. 각 설치 파일은 다음에 접속하시면
됩니다.
<br>

* `Java 11`: Oracle -> Products -> Java -> Download Java button 클릭 -> 아래쪽에 Java 11 tab 선택후
  다운로드(Oracle login이 필요합니다.)
* Eclipse: Eclipse -> Eclipse IDE 2022‑06 다운로드
<br>

먼저 기본에 설치되어 있던 `Java`를 삭제한 후 다시 `Java 11`버전으로 설치하시면 됩니다.
<br><br>

Eclipse는 기존에 설치되어 있던 폴더만 삭제해주면 됩니다. 아마 `c:/user/사용자이름`경로에 다음과 같은 폴더가 있을텐데
이들만 삭제해주고 다시 설치하도록 하죠
<br>

* `eclipse`
* `eclipse-workspace`
* `.eclipse`
* `.p2`
<br>

설치가 끝났으면 Eclipse를 실행하고 이제 간단한 코드 부터 시작해 보도록 하겠습니다.

## 객체지향의 기본적인 개념

먼저 프로그래밍 패러다임에 대해서 알아보도록 하죠. 프로그래밍 패러다임은 아주쉽게 생각하면
프로그래밍 방법론 이라고 생각하시면 됩니다. 어떤 방법으로 프로그래밍을 하느냐 하는 것이죠.
<br><br>

많은 방법론이 있겠지만 대표적인 것으로는 다음과 같은 것들이 있습니다.
<br>

* 함수형 프로그래밍 -> 구조적 프로그래밍, 절차적 프로그래밍
* 객체지향 프로그래밍
* 선언적 프로그래밍
<br>

이런 방법론이 있는데 가장 대표적인 두가지 방법에 대해 알아보도록 하겠습니다.

### 절차적 프로그래밍

절차적 프로그래밍의 특징은 다음과 같습니다.
<br>

* 프로그램을 모듈(함수)이라는 실행단위로 분할.
* 상위 모듈에서 하위 모듈을 호출하는 방식으로 프로그램을 구성
* 프로그램 작성 시 기능으로 세분화 한 다음 각각의 기능을 모듈로 제작
* 프로그램의 구조를 이해하기 쉽고 빠르게 프로그램을 작성할 수 있다.
* 프로그램 규모가 커지게 되면 유지보수와 기존 코드의 재사용에 한계가 나타나게 된다.

### 객체지향 프로그래밍

* 현실세계의 해결해야 하는 문제를 그대로 프로그램으로 구현.
* 즉, 프로그램을 기능으로 세분화 하지 않고 프로그램을 구성하는 주체(객체)들을 파악하고
그 객체들간의 data 흐름에 초점을 맞추어서 프로그램 작성
* 프로그램의 설계, 작성이 상대적으로 어려움.
* 이렇게 작성된 객체지향 프로그램은 유지보수와 재사용성에 이점이 있다.
<br>

간단하게 은행 프로그램을 예로 들어보면 될 듯 합니다.
<br><br>

객체지향 프로그램은 다음과 같은 방법을 이용합니다.
<br>

* 현실세계의 해결해야 하는 문제를 그대로 프로그램으로 표현.
* 해결해야 하는 문제들을 구성하고 있는 객체들을 파악
* 해당 객체의 `상태`와 `행위` 규정
* 이렇게 규정된 해당 객체의 내용을 프로그램적으로 표현
* 이때 사용되는 것이 class.
<br>

결국 class는 객체 모델링의 수단이라고 할 수 있습니다. 또한 instance를 만들어내는 수단이기도
하며 추상 데이터 타입(`ADT`)이기도 합니다.
<br><br>

추상 데이터 타입은 다음과 같이 설명할 수 있습니다.
<br>

* 사용자 정의 data type으로 사용.
* 추상 데이터 타입, reference data type 이라고도 한다.
* 기존 data type을 이용하여 새로운 data type을 만드는 작업.

## Java 프로그램의 특징

위에서 살펴본것처럼 Java 프로그램은 class로 구성됩니다. 우리가 class를 작성한 후
이를 `javac.exe`로 컴파일 하고 이를 `java.exe`로 실행하는 것이죠.
<br><br>

`eclipse`는 명령어로 수행해야 할 이런 작업을 편하게 할 수 있도록 도와주는 IDE입니다.

## Java 프로그램의 실행

`Java` 프로그램의 실행은 다음과 같은 순서를 거칩니다.
<br>

1. 프로그램 실행에 필요한 `class`를 `class loader`를 이용해 loading합니다.
2. 각 class 파일을 검사합니다.(파일형식, 악성코드 체크)
3. 지정된 class에서 `main()`을 호출합니다.
<br>

`main()`의 첫줄부터 코드가 실행되기 시작하여 마지막 코드까지 모두 실행되면 프로그램이 종료되고
프로그램에서 사용했던 자원들이 모두 반납됩니다.
